Immutable Version Control

Treating copy like code. The post_versions table tracks every generation cycle.

When you click "Regenerate" or "Correct", the system doesn't overwriteâ€”it branches. You can traverse the history of a post, compare iterations, and revert to any previous state with zero data loss.

LLM Agnostic Core

Built on top of OpenRouter, giving you access to the entire model landscape.

Swap models per task: Use Claude 3.5 Sonnet for creative writing, GPT-4o for reasoning, or Llama 3 for local privacy. You are not locked into a single vendor's API or pricing model.

Webhook Orchestration

We decouple generation from publication. The PublisherService pushes approved content to a centralized webhook (Make.com/Zapier).

This allows for infinite downstream flexibility: route content to Slack for final approval, push to a custom CMS, or trigger complex automation flows without modifying the core codebase.

WORKFLOW

From Brief to Broadcast

The "Brief" First Model

Most tools start with a "Post". We start with a "Brief". A Brief is a strategic container that holds your campaign's core message, assets, and links. The system then "explodes" this Brief into platform-native assets for X, LinkedIn, Instagram, and more simultaneously.

Secure OAuth 2.0

Enterprise-grade token management. Your credentials are encrypted and never leave your self-hosted instance.

Draft & Queue

Separate "Drafting" from "Scheduling". Review content in a dedicated workspace before committing to the publish queue.
